# DDD 领域模型设计

## 项目信息
- **项目名称**: 工程图纸设计助手
- **技术栈**: Qt 5.15.2 + C++ 17
- **设计方法**: 领域驱动设计 (DDD)
- **创建日期**: 2026-03-24
- **最后更新**: 2026-03-31

---

## 1. 领域概述

### 1.1 领域定义
工程图纸设计助手是一个基于对话的智能辅助系统，通过自然语言交互帮助用户解决各类工程图纸设计问题。系统支持加载知识库、管理和使用 Skill 扩展，并提供 Markdown 格式的内容渲染能力。新增知识图谱可视化和 RAG 智能回答能力。

### 1.2 领域愿景
成为工程师的智能设计伙伴，通过自然语言交互提供工程图纸设计支持，包括规范查询、图纸审查、设计建议等能力。通过知识图谱直观展示概念关系，通过 RAG 技术结合知识库内容提供更准确的回答。

### 1.3 核心价值
- **便捷性**: 通过对话方式快速获取设计支持
- **专业性**: 基于工程知识库提供准确的设计建议
- **可视化**: Markdown 渲染提供清晰的内容展示，知识图谱直观展示概念关系
- **智能化**: RAG 技术自动结合知识库内容生成回答
- **可扩展性**: 通过 Skill 机制灵活扩展设计能力

## 3. 边界上下文

### 3.1 ChatContext（聊天上下文）

**职责**: 管理用户与助手之间的对话交互，支持 RAG 智能回答和优化的聊天气泡布局。

**核心概念**:
- **ChatSession（聊天会话）**: 聚合根，表示一次完整的对话会话
- **Message（消息）**: 实体，表示会话中的一条消息
- **ChatProcessor（对话处理器）**: 领域服务，处理消息流转，支持 RAG 上下文解析

**核心契约**:
```cpp
class ChatSession {
    void addMessage(const Message& msg);
    void clearHistory();
    std::vector<Message> getRecentMessages(int count);
    MessageId generateMessageId();
}

enum class MessageType {
    User,
    Assistant,
    System
};

// ChatProcessor 新增 RAG 支持
class ChatProcessor {
    Message processUserMessage(const Message& userMessage);
    std::string generateResponse(const std::string& query);
private:
    std::string extractKnowledgeBaseContext(const std::string& query);
    std::string extractUserQuestion(const std::string& query);
};
```

### 3.2 KnowledgeContext（知识库上下文）

**职责**: 管理工程图纸设计知识库的加载和查询，支持全文检索、切片管理和知识图谱可视化。

**核心概念**:
- **KnowledgeBase（知识库）**: 聚合根，表示一个知识库
- **Document（文档）**: 实体，表示知识库中的文档
- **Chunk（切片）**: 值对象，表示文档的分块内容
- **KnowledgeLoader（知识加载器）**: 领域服务，加载知识库文件
- **GraphBuilder（图谱构建器）**: 领域服务，构建知识图谱数据

**核心契约**:
```cpp
class KnowledgeBase {
    void addDocument(const Document& doc);
    void removeDocument(const DocumentId& id);
    std::vector<Document> searchDocuments(const std::string& keyword);
    std::vector<Chunk> searchChunks(const std::string& keyword);
    bool isLoaded() const;
}

// 切片结构
struct Chunk {
    ChunkId id;
    std::string content;
    int position;
    int size;
    std::vector<std::string> keywords;
};

// 知识图谱节点
struct GraphNode {
    std::string id;
    std::string label;
    int freq;
    double x, y;
    double r;
};

// 知识图谱边
struct GraphEdge {
    std::string from, to;
    double w;
};

enum class KnowledgeType {
    DesignStandard,
    DrawingSpecification,
    ProcessGuide
};
```

### 3.3 SkillContext（技能上下文）

**职责**: 管理 Skill 的注册、加载、执行和生命周期

**核心概念**:
- **Skill（技能）**: 实体，表示一个可执行的设计能力扩展
- **SkillRegistry（技能注册表）**: 聚合根，管理所有已注册的 Skill
- **SkillExecutor（技能执行器）**: 领域服务，执行 Skill 并返回结果

**核心契约**:
```cpp
class SkillRegistry {
    void registerSkill(const Skill& skill);
    void unregisterSkill(const SkillId& id);
    Skill* getSkill(const SkillId& id);
    std::vector<Skill> getAllSkills();
}

class Skill {
    SkillId id;
    std::string name;
    std::string description;
    std::function<ExecuteResult(const Parameters&)> execute;
}
```

### 3.4 RenderingContext（渲染上下文）

**职责**: Markdown 内容的解析和渲染为可展示格式

**核心概念**:
- **MarkdownDocument（Markdown 文档）**: 实体，表示 Markdown 文本
- **RenderResult（渲染结果）**: 值对象，表示渲染后的内容
- **MarkdownRenderer（渲染器）**: 领域服务，执行 Markdown 到 HTML 的转换

**核心契约**:
```cpp
class MarkdownRenderer {
    RenderResult render(const std::string& markdown);
    std::string toHtml(const RenderResult& result);
}
```

### 3.5 AssistantContext（助手上下文）

**职责**: 工程图纸设计助手的核心业务逻辑，协调其他上下文完成设计辅助任务

**核心概念**:
- **DesignAssistant（设计助手）**: 聚合根，表示助手实例
- **QueryResolver（查询解析器）**: 领域服务，解析用户查询
- **DesignAnalyzer（设计分析器）**: 领域服务，分析设计问题

**核心契约**:
```cpp
class DesignAssistant {
    MessageResponse processUserQuery(const UserQuery& query);
    void loadKnowledgeBase(const std::string& path);
    void registerSkill(const Skill& skill);
}

class QueryResolver {
    QueryType resolve(const std::string& query);
    std::vector<std::string> extractKeywords(const std::string& query);
}
```

## 6. 领域事件

### 6.1 事件定义
```cpp
struct DomainEvent {
    virtual ~DomainEvent() = default;
    virtual std::string eventType() const = 0;
    QDateTime timestamp;
};

// 消息接收事件
struct MessageReceivedEvent : public DomainEvent {
    MessageId messageId;
    std::string content;
    std::string eventType() const override { return "MessageReceived"; }
};

// 知识库加载完成事件
struct KnowledgeLoadedEvent : public DomainEvent {
    KnowledgeId knowledgeId;
    int documentCount;
    std::string eventType() const override { return "KnowledgeLoaded"; }
};

// Skill 注册成功事件
struct SkillRegisteredEvent : public DomainEvent {
    SkillId skillId;
    std::string skillName;
    std::string eventType() const override { return "SkillRegistered"; }
};

// Skill 执行完成事件
struct SkillExecutedEvent : public DomainEvent {
    SkillId skillId;
    bool success;
    std::string eventType() const override { return "SkillExecuted"; }
};
```

## 7. 领域服务

### 7.1 ChatProcessor
- 处理用户输入消息
- 协调知识库和 Skill 系统生成响应
- 管理对话上下文和状态

### 7.2 KnowledgeLoader
- 从文件系统加载知识库文档
- 解析文档内容并建立索引
- 支持增量更新知识库

### 7.3 SkillExecutor
- 验证 Skill 参数
- 执行 Skill 逻辑
- 收集和格式化执行结果

### 7.4 MarkdownRenderer
- 解析 Markdown 语法
- 生成 HTML 格式内容
- 处理代码高亮和图片链接

### 7.5 GraphRenderer（新增）
- **力导向布局计算**：节点斥力 + 边引力 + 中心引力
- **原生 Qt 绘制**：使用 QPainter 直接绘制节点和边
- **交互支持**：鼠标拖拽平移、滚轮缩放、悬停提示

### 7.6 QueryResolver
- 分析用户查询意图
- 提取查询关键词
- 匹配相应的 Skill 或知识库

## 10. 设计原则

### 10.1 战略设计原则
- **按业务边界划分上下文**: 每个上下文专注于一个清晰的业务领域
- **上下文自治**: 每个上下文独立维护自己的模型和规则
- **最小化上下文间依赖**: 通过防腐层和接口实现松耦合

### 10.2 战术设计原则
- **聚合根保护一致性**: 通过聚合根控制内部实体的访问和修改
- **领域逻辑封装**: 业务逻辑放在领域模型中，不泄露到应用层
- **值对象不可变**: 值对象创建后不可修改，确保线程安全

### 10.3 实现原则
- **依赖倒置**: 高层模块依赖抽象，不依赖具体实现
- **单一职责**: 每个类只负责一个清晰的职责
- **开闭原则**: 对扩展开放，对修改关闭
