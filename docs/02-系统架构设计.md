# 系统架构设计

## 项目信息

- **项目名称**: 工程图纸设计助手
- **技术栈**: Qt 5.15.2 + C++ 14
- **设计方法**: DDD + 分层架构
- **创建日期**: 2026-03-24

---

## 1. 架构概述

### 1.1 架构风格

采用 **DDD（领域驱动设计）+ 分层架构**，结合 **事件驱动** 和 **依赖注入** 的设计风格。

### 1.2 架构目标

- **可维护性**: 清晰的分层和职责划分，便于理解和维护
- **可扩展性**: 通过 Skill 机制和插件化设计支持功能扩展
- **可测试性**: 层间通过接口交互，支持单元测试和集成测试
- **简洁性**: 代码简洁，逻辑清晰，避免过度设计

### 1.3 架构约束

- Qt 版本: 5.15.2
- C++ 标准: C++11
- 无第三方重型依赖，保持轻量化

## 2. 分层架构设计

### 2.1 分层定义

```
┌──────────────────────────────────────────────────────────┐
│                  Presentation Layer                      │
│                   (表现层 - UI)                           │
│  - ChatWindow、SkillPanel、KBManager、Settings           │
└────────────────────┬─────────────────────────────────────┘
                     │ uses
┌────────────────────▼─────────────────────────────────────┐
│                 Application Layer                        │
│                 (应用层 - 协调)                          │
│  - ChatAppService、SkillAppService、KBAppService         │
└────────────────────┬─────────────────────────────────────┘
                     │ depends
┌────────────────────▼─────────────────────────────────────┐
│                  Domain Layer                            │
│               (领域层 - 核心业务)                        │
│  - ChatContext、KnowledgeContext、SkillContext           │
│  - RenderingContext、AssistantContext                    │
└────────────────────┬─────────────────────────────────────┘
                     │ implements
┌────────────────────▼─────────────────────────────────────┐
│               Infrastructure Layer                       │
│              (基础设施层 - 技术实现)                      │
│  - FileStorage、QtUIUtils、MarkdownParser、EventBus      │
└──────────────────────────────────────────────────────────┘
```

### 2.2 各层职责

#### 2.2.1 表现层（Presentation Layer）

**职责**: 负责用户界面的展示和交互

**核心组件**:

- **ChatWindow**: 主聊天窗口

  - 显示消息列表（支持 Markdown 渲染）
  - 用户输入框
  - 发送按钮
  - 清空历史按钮
- **SkillPanel**: Skill 管理面板

  - 显示已安装的 Skill 列表
  - Skill 信息查看
  - Skill 启用/禁用切换
  - Skill 参数配置
- **KBManager**: 知识库管理窗口

  - 加载知识库文件
  - 显示知识库文档列表
  - 知识库搜索功能
  - 知识库状态查看
- **Settings**: 设置窗口

  - 主题设置
  - 字体大小
  - 知识库路径配置
  - Skill 目录配置

**技术特点**:

- 使用 Qt Widgets 实现界面
- WebView 或 QTextBrowser 渲染 Markdown
- 信号槽机制处理用户交互
- 异步加载知识库和 Skill

#### 2.2.2 应用层（Application Layer）

**职责**: 协调领域对象完成业务用例，不包含业务逻辑

**核心组件**:

- **ChatAppService**: 聊天应用服务

  - 发送消息
  - 获取历史消息
  - 清空会话
- **SkillAppService**: Skill 应用服务

  - 安装 Skill
  - 卸载 Skill
  - 获取 Skill 列表
  - 执行 Skill
- **KBAppService**: 知识库应用服务

  - 加载知识库
  - 卸载知识库
  - 搜索知识库
- **RenderAppService**: 渲染应用服务

  - 渲染 Markdown
  - 生成 HTML

**设计原则**:

- 每个应用服务对应一个业务用例
- 方法命名反映业务意图
- 协调多个领域对象完成事务
- 处理事务边界

#### 2.2.3 领域层（Domain Layer）

**职责**: 包含核心业务逻辑和领域模型

**核心组件**:

- **ChatContext**: 聊天上下文

  - ChatSession（聚合根）
  - Message（实体）
  - ChatProcessor（领域服务）
- **KnowledgeContext**: 知识库上下文

  - KnowledgeBase（聚合根）
  - Document（实体）
  - KnowledgeLoader（领域服务）
- **SkillContext**: 技能上下文

  - SkillRegistry（聚合根）
  - Skill（实体）
  - SkillExecutor（领域服务）
- **RenderingContext**: 渲染上下文

  - MarkdownDocument（实体）
  - RenderResult（值对象）
  - MarkdownRenderer（领域服务）
- **AssistantContext**: 助手上下文（核心）

  - DesignAssistant（聚合根）
  - QueryResolver（领域服务）
  - DesignAnalyzer（领域服务）

**设计原则**:

- 领域逻辑封装在领域模型中
- 不依赖任何基础设施
- 通过领域服务实现跨聚合的业务逻辑
- 使用领域事件实现松耦合

#### 2.2.4 基础设施层（Infrastructure Layer）

**职责**: 提供技术支持，实现领域层定义的接口

**核心组件**:

- **FileStorage**: 文件存储

  - JSON 格式存储会话数据
  - 文件系统管理知识库和 Skill
  - 配置文件读写
- **QtUIUtils**: Qt UI 工具

  - 对话框工具
  - 消息提示
  - 文件选择器
- **MarkdownParser**: Markdown 解析器

  - 集成轻量级 Markdown 解析库（如 cmark-gfm）
  - 生成 HTML
  - 代码高亮支持
- **EventBus**: 事件总线

  - 订阅/发布领域事件
  - 异步事件处理
  - 事件持久化

**设计原则**:

- 实现领域层定义的接口
- 封装第三方库
- 提供技术能力给上层使用
- 保持与业务无关

## 4. 依赖关系

### 4.1 依赖规则

```
表现层 ──depends──> 应用层 ──depends──> 领域层
                                   ↑
                                   │ implements
                                   │
                             基础设施层
```

### 4.2 依赖注入

使用构造函数注入（Constructor Injection）方式实现依赖倒置：

```cpp
// 应用层依赖领域层抽象
ChatAppService::ChatAppService(
    std::shared_ptr<ChatSessionRepository> chatRepo,
    std::shared_ptr<KnowledgeBaseRepository> knowledgeRepo,
    std::shared_ptr<SkillRegistry> skillRegistry,
    std::shared_ptr<QueryResolver> queryResolver
) : m_chatRepo(chatRepo),
    m_knowledgeRepo(knowledgeRepo),
    m_skillRegistry(skillRegistry),
    m_queryResolver(queryResolver) {}

// 基础设施层实现领域层接口
class FileStorage : public ChatSessionRepository { ... };
```

### 4.3 工厂模式

使用工厂模式创建应用服务：

```cpp
class ServiceFactory {
public:
    static std::shared_ptr<ChatAppService> createChatAppService(
        const std::string& dataDir
    ) {
        auto chatRepo = std::make_shared<FileStorage>(dataDir);
        auto knowledgeRepo = std::make_shared<FileStorage>(dataDir);
        auto skillRegistry = std::make_shared<SkillRegistry>();
        auto queryResolver = std::make_shared<QueryResolver>();

        return std::make_shared<ChatAppService>(
            chatRepo, knowledgeRepo, skillRegistry, queryResolver
        );
    }
};
```

## 5. 事件驱动架构

### 5.1 事件总线

```cpp
class EventBus {
public:
    // 订阅事件
    template<typename EventType>
    void subscribe(std::function<void(const EventType&)> handler);

    // 发布事件
    void publish(std::shared_ptr<DomainEvent> event);

    // 异步发布
    void publishAsync(std::shared_ptr<DomainEvent> event);

private:
    std::unordered_map<std::string, std::vector<std::function<void(std::shared_ptr<DomainEvent>)>>> m_handlers;
};
```

### 5.2 事件流

```
用户发送消息
    ↓
ChatAppService::sendUserMessage()
    ↓
ChatSession::addMessage()
    ↓
发布 MessageReceivedEvent
    ↓
事件总线分发到订阅者
    ↓
QueryResolver::handleMessageReceivedEvent()
    ↓
SkillExecutor::handleMessageReceivedEvent()
```

class ChatSession {

        QMutexLocker locker(&m_mutex);

    }

private:
    mutable QMutex m_mutex;
    std::vector<Message> m_messages;

## 13. 技术决策记录

| 决策          | 选项                  | 选择       | 原因                      |
| ------------- | --------------------- | ---------- | ------------------------- |
| UI 框架       | Qt Widgets / Qt Quick | Qt Widgets | 更稳定，Qt 5.6.3 原生支持 |
| Markdown 渲染 | cmark-gfm / hoedown   | cmark-gfm  | 功能完整，GFM 兼容        |
| 数据存储      | JSON / SQLite         | JSON       | 简单，易于调试            |
| 知识库格式    | 纯文本 / Markdown     | Markdown   | 支持格式化                |
| Skill 格式    | DLL / Shared Library  | DLL        | Windows 兼容性好          |
