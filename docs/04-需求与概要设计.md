## Part 1  产品需求（PRD）

### 1. 项目概述

#### 1.1 项目背景
工程师在绘图、审图过程中需要频繁查询设计规范和工程标准。传统查阅方式效率低下，需要翻阅纸质手册或在多个文档中搜索。通过自然语言对话 + 知识库检索，可以显著提升工程师效率。

#### 1.2 项目目标
基于 Qt 5.15.2 + C++14 开发桌面应用，实现：
1. 自然语言对话交互，解答工程图纸设计问题
2. 加载和管理工程知识库（标准/规范/流程），支持网页链接导入
3. 支持加载、管理和使用 Skill 扩展功能，支持对话式创建新技能
4. Markdown 格式内容渲染
5. 界面清爽、操作简洁

#### 1.3 目标用户
- 机械工程师、电气工程师、建筑工程师
- 需要绘制和审查工程图纸的技术人员

#### 1.4 非功能需求

| 指标 | 要求 |
|------|------|
| 消息响应 | < 2 秒 |
| 知识库加载 | < 5 秒 |
| 知识库搜索 | < 1 秒 |
| 内存占用（空闲） | < 500 MB |
| 操作系统 | Windows 7+ / Linux / macOS 10.12+ |
| 连续运行 | 无崩溃 > 24 小时 |

### 2. 功能需求

#### FR-001  聊天对话（P0）
- 文本输入，Enter 发送，Shift+Enter 换行
- 用户消息右对齐（蓝色气泡 + 头像），助手回复左对齐（白色气泡 + 头像）
- 消息时间戳，自动滚动到最新
- Markdown 渲染（标题/列表/代码块/粗体/斜体）
- 支持 `@skill_id 参数=值` 格式调用技能
- 清空会话

#### FR-002  知识库管理（P0）
- 列出并加载 `data/knowledge/` 下所有知识库目录
- 关键词全文检索，展示摘要
- 一键卸载
- **🌐 导入网址**：输入 URL 自动抓取正文，保存为 Markdown，热更新到已加载知识库

#### FR-003  Skill 管理（P1）
- 列出 `data/skills/` 下所有已安装技能（读取 skill.json）
- 展示技能详情（名称/版本/描述/参数列表）
- 从目录批量加载
- **✨ 对话创建**：两步向导（基本信息 + 参数表），生成 skill.json 并热注册
- `@skill_id` 调用，参数通过键值对传递

#### FR-004  Markdown 渲染（P0）
- 支持标准 Markdown 语法（标题、列表、代码块、粗体、斜体、分隔线）
- 代码块带背景色区分
- 链接可点击跳转

#### FR-005  设置（P2，待实现）
- 主题切换（浅色/深色）
- 知识库/Skill 路径配置
- 字体大小调整

#### FR-006  日志（P2，待实现）
- 操作日志记录与查看

## Part 2  概要设计

### 5. 系统架构

#### 5.1 分层架构

```
┌────────────────────────────────────────┐
│     Presentation Layer（表现层）        │
│  ChatWidget / SkillPanel / KBPanel     │
├────────────────────────────────────────┤
│     Application Layer（应用层）         │
│  ChatAppService / SkillAppService      │
│  KBAppService                          │
├────────────────────────────────────────┤
│       Domain Layer（领域层）            │
│  ChatSession / KnowledgeBase           │
│  SkillRegistry / MarkdownRenderer      │
├────────────────────────────────────────┤
│   Infrastructure Layer（基础设施层）    │
│  SkillLoader / KnowledgeLoader         │
│  SimpleMarkdownRenderer / QNetwork     │
└────────────────────────────────────────┘
```

依赖方向：Presentation → Application → Domain ← Infrastructure

#### 5.2 模块划分

| 模块 | 层级 | 职责 |
|------|------|------|
| main_window | Presentation | 主窗口，左侧导航栏 + QStackedWidget |
| chat_widget | Presentation | 聊天面板，消息渲染和输入 |
| skill_panel | Presentation | 技能面板，列表/详情/创建向导 |
| kb_panel | Presentation | 知识库面板，加载/搜索/网页导入 |
| ChatAppService | Application | 解析 @指令，路由到技能或知识库 |
| SkillAppService | Application | 技能加载/注册/执行 |
| KBAppService | Application | 知识库加载/搜索/URL 导入 |
| ChatSession | Domain | 消息历史管理 |
| ChatProcessor | Domain | 对话处理逻辑 |
| KnowledgeBase | Domain | 文档集合，全文检索 |
| SkillRegistry | Domain | 技能注册表 |
| SkillExecutor | Domain | 技能执行 |
| SimpleMarkdownRenderer | Infrastructure | 轻量 Markdown → HTML |
| SkillLoader | Infrastructure | 解析 skill.json |
| KnowledgeLoader | Infrastructure | 读取 .md 文件 |

### 6. 核心流程

#### 6.1 发送消息流程
```
用户输入 → ChatWidget::onSend()
    → ChatAppService::handleUserInput()
        → 检测 @skill_id 指令 → SkillExecutor::execute()
        → 否则 → ChatProcessor::process()（结合知识库检索）
    → 回调 appendHtmlReply()
    → QTextBrowser 渲染
```

#### 6.2 加载知识库流程
```
KBPanel 选中知识库 → KBAppService::loadKnowledgeBase()
    → KnowledgeLoader::load()（读取目录下所有 .md）
    → KnowledgeBase 构建文档索引
    → ChatAppService 绑定当前知识库
```

#### 6.3 网页导入流程
```
用户输入 URL → KBAppService::importFromUrl()
    → QNetworkAccessManager 同步抓取（QEventLoop）
    → HTML 提取正文（去 script/style/标签）
    → 生成 Markdown 文件保存到知识库目录
    → KnowledgeBase::addDocument() 热更新
```

#### 6.4 创建技能流程
```
SkillCreateWizard 两步填写
    → 生成 skill.json 写入 data/skills/<id>/
    → SkillAppService::loadSkill()
    → SkillRegistry::registerSkill() 热注册
```

### 7. 数据模型

#### 7.1 skill.json 格式
```json
{
  "id": "drawing-review",
  "name": "图纸审查",
  "version": "1.0.0",
  "description": "自动审查工程图纸，检查尺寸标注规范性",
  "author": "工程团队",
  "parameters": [
    {
      "name": "part",
      "type": "string",
      "required": true,
      "description": "零件名称"
    }
  ]
}
```

#### 7.2 知识库目录结构
```
data/knowledge/<库名>/
├── doc1.md
├── doc2.md
└── ...
```

### 8. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 网页抓取失败 | 弹出 QMessageBox，显示错误原因分类 |
| 知识库目录不存在 | 提示用户检查路径 |
| skill.json 格式错误 | 跳过该技能，控制台输出警告 |
| 未加载知识库时搜索 | 结果区显示引导提示 |
| 未加载知识库时导入网址 | 提示先加载知识库 |

---

### 9. 并发与性能

- 当前版本：同步处理（主线程），适合单用户桌面场景
- 网页导入使用 `QEventLoop` 实现"伪同步"（不阻塞事件循环）
- 知识库全文检索：线性遍历 + `std::string::find`，文档量大时可升级为倒排索引
- 后续优化方向：`QtConcurrent::run` 异步加载知识库，避免 UI 卡顿
