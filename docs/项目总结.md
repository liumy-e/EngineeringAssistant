# 工程图纸设计助手 - 项目总结

**项目名称**: 工程图纸设计助手
**技术栈**: Qt 5.15.2 + C++14
**设计方法**: DDD（领域驱动设计）+ 分层架构
**最后更新**: 2026-03-25

---

## 项目目标

开发一个基于 Qt 5.15.2 + C++ 的桌面应用，实现以下核心功能：
1. 自然语言对话交互，解答工程图纸设计问题
2. 加载和管理工程知识库（设计标准、规范、流程等），支持网页链接导入
3. 支持加载、管理和使用 Skill 扩展功能，支持对话式创建新技能
4. Markdown 格式内容渲染，提供清晰的信息展示
5. 界面清爽，深色导航栏 + 内容区切换布局

## 已实现功能

### 1. 聊天对话
- 用户消息右对齐蓝色气泡 + 头像，AI 回复左对齐白色气泡 + 头像
- Enter 发送，Shift+Enter 换行
- 消息时间戳、自动滚动到最新
- 清空会话按钮

### 2. 知识库管理
- 列出 `data/knowledge/` 下所有知识库，一键加载/卸载
- 关键词全文检索，高亮展示匹配结果
- **🌐 导入网址**：输入 URL 自动抓取网页正文，生成 Markdown 文档并热更新到知识库

### 3. Skill 管理
- 从目录批量加载 skill.json，展示技能列表和详情
- **✨ 对话创建**：两步向导（基本信息 + 参数列表），生成 skill.json 并热注册
- 内置技能：图纸审查、尺寸检查、标准查询
- `@skill_id` 指令在聊天中直接调用技能

### 4. Markdown 渲染
- 内置轻量 Markdown 渲染器（标题/列表/代码块/粗体/斜体/分隔线）
- 代码块带背景色区分

### 5. UI 布局
- 左侧深色导航栏（💬 对话 / 🔧 技能 / 📚 知识库），图标+文字竖排
- QStackedWidget 内容区无闪烁切换
- 各面板统一顶部标题栏风格


## 常见问题

### Q1: 运行时找不到 Qt 库？
```bash
export DYLD_FRAMEWORK_PATH=~/Qt/5.15.2/clang_64/lib
export DYLD_LIBRARY_PATH=~/Qt/5.15.2/clang_64/lib
./EngineeringAssistant
```

### Q2: 如何添加新技能？

**方法一（推荐）**：在技能面板点「✨ 对话创建」，按向导填写信息自动生成。

**方法二（手动）**：
1. 在 `data/skills/<id>/` 下创建 `skill.json`：
```json
{
  "id": "my-skill",
  "name": "我的技能",
  "version": "1.0.0",
  "description": "技能描述",
  "author": "作者",
  "parameters": [
    { "name": "param1", "type": "string", "required": true, "description": "参数描述" }
  ]
}
```
2. 在技能面板点「加载目录」重新加载。

### Q3: 如何添加知识库？

**方法一**：在知识库面板点「🌐 导入网址」，输入网页 URL 自动抓取导入。

**方法二（本地文件）**：
1. 在 `data/knowledge/<库名>/` 下放入 `.md` 文档
2. 在知识库面板选中并点「加载」

### Q4: 聊天中如何调用技能？
在输入框输入 `@技能ID 参数名=参数值`，例如：
```
@drawing-review part=主轴零件
@standard-query standard=GB/T14689
```
